## About Distro

<img width="732" height="197" alt="image" src="https://github.com/user-attachments/assets/57194ac8-7ea5-447b-bb89-42d7c46c749b" />

Distro is a dynamic system tailored for users who seek sophisticated report distribution capabilities within IBM Planning Analytics. It enables the exportation of TM1Web, UX, and PAW reports by dimension, offering diverse integration and distribution methods to suit various user needs:

- Targeted Report Exportation: Tailor your report distribution by specific dimensions across TM1Web, UX, and PAW platforms.
- Flexible Distribution Options: Choose how and where your reports are distributed, accommodating a range of user preferences and requirements.
- Streamlined User Experience: Benefit from a system designed with the end-user in mind, facilitating ease of use and efficiency in report management.

## How to submit a enhancement request or bug

1. Click on the Issues tab at the top of this page.
2. **BEFORE** submitting a new issue please search the current issues to see if it already exists.
3. Click on **New issue** button.
4. For the **Title** line enter either a description of the bug or a description of what you want added to Distro.
5. For bugs the following details must be entered:
   * Version of Distro (get this from the About Page in the Distro)
   * Version of TM1 (get this from the Pulse Dashboard)
   * **Step by step details of how to replicate the issue**
   * Provide as many screenshots as possible.
6. Click on **Submit new issue**.

## Public/Private profiles and schedules

#### **Profiles**

* Every user with profile_write capbility can create a profile (default private profile)\\
* User without admin_write capbility can only see their profiles(private and public)\\
* User without admin_write capbility can only edit their private profiles\\
* User with admin_write capbility can see their profiles and all the public profiles\\
* User with admin_write capbility can turn other person's public profiles into private\\
* User with admin_write capbility edit other person's public profile\\
* Only user with admin_write capbility can toggle between public/private profile\\
* Public profile can not turn to private if it's used in one of the public schedule

#### **Schedules**

* Every user with schedule_write capbility can create a schedule (default private schedule)\\
* Only profiles displayed in the user profile list can be add into schedule\\
* User without admin_write capbility can only see their schedules(private and public)\\
* User without admin_write capbility can only edit their private schedules\\
* User with admin_write capbility can see their schedules and all the public schedules\\
* User with admin_write capbility can turn other person's public schedules into private\\
* User with admin_write capbility edit other person's public schedules\\
* Only user with admin_write capbility can toggle between public/private schedules\\
* Only public profiles can be included into public schedule
