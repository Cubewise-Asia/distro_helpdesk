# Contributing to Distro Helpdesk

Thank you for helping improve **Distro**. This repository is the central helpdesk for **bug reports**, **feature requests / wishes**, and **questions**.

## Quick Steps
- Search existing issues before opening a new one.
- Use the provided templates for bug reports and feature requests.
- Add screenshots, logs, and exact reproduction steps whenever possible.

## Issue Types
- **Bug report:** Describe the problem, steps to reproduce, expected vs actual result, and environment details.
- **Feature request / wish:** State the use case, desired behavior, and any examples or mockups.
- **Question:** If unsure whether behavior is expected, open a Question issue.

## Submitting a Pull Request
1. Discuss changes in an issue first (if unsure).
2. Fork and create a feature branch.
3. Keep changes scoped and add context in the PR template.
4. Ensure documentation and templates stay accurate.

## Community Expectations
- Be respectful and collaborative (see `CODE_OF_CONDUCT.md`).
- Provide clear, reproducible information.
- Respond to follow-up questions to help triage faster.

## Thank You
Your feedback and contributions help Distro evolve. We appreciate your participation.
